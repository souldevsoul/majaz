THE 13 CUSTOM RULES (by category):
1. Link Validation (1 rule)
   - no-broken-internal-links: Detects 404 errors before deploy

2. Brand Consistency (2 rules)
   - use-styleguide-colors-only: Enforce approved color palette
   - consistent-company-info: Single email, company name

3. Content Consistency (2 rules)
   - consistent-payment-providers: One payment method
   - (Email/company info validation)

4. UX Consistency (6 rules)
   - no-button-without-handler: Buttons must do something
   - no-form-without-submit: Forms must handle submission
   - no-missing-alt-text: Images need alt text (accessibility)
   - no-generic-placeholders: Specific placeholder text
   - require-loading-state-on-async-button: Show feedback during async
   - require-aria-label-on-icon-buttons: Icon buttons accessible

5. Error Handling & Quality (2 rules)
   - require-try-catch-fetch: API calls must handle errors
   - require-empty-state: Lists need "no data" message

6. Performance (1 rule)
   - require-image-optimization: Use Next.js Image component

RECOMMENDED FOR MAJAZ (Priority Order)
=====================================

CRITICAL (ERROR level - zero tolerance):
1. use-styleguide-colors-only
   - Colors: Black (#111111), Gold (#D4AF37), Ivory (#FFFFF0)
   - Prevents off-brand color experimentation
   - Smart: Ignores non-color utilities (text-xs, bg-gradient)

2. consistent-company-info
   - Email: support@majaz.ae
   - Company: MAJAZ
   - Prevents brand variation accidents

3. no-broken-internal-links
   - Luxury brand = no 404 errors before deploy
   - Handles Next.js [locale], (auth), (marketing) patterns

4. no-missing-alt-text
   - Accessibility requirement
   - SEO for vehicle listings
   - Screen reader support

5. no-button-without-handler
   - All CTAs must function
   - "Request Assessment" button must work

6. no-form-without-submit
   - Assessment form must submit
   - Payment form must process

7. require-aria-label-on-icon-buttons
   - RTL (Arabic) accessibility
   - Screen readers need labels

HIGH VALUE (WARN level - improve quality):
8. no-generic-placeholders
   - "Enter vehicle link" not "Enter text"
   - Shows attention to detail

9. require-loading-state-on-async-button
   - Payment buttons show loading
   - Assessment requests show feedback

10. require-try-catch-fetch
    - API failures handled gracefully
    - Auction scrapers, Perplexity API, OpenAI calls

11. require-empty-state
    - No empty lists without message
    - "No assessments yet" message

12. require-image-optimization
    - Next.js Image for hero images
    - Performance for mobile users
